I lead AI product engineering at YTL AI Labs — where I've worked on the org's public conversational-AI product and now sit on a confidential internal project — and I set the engineering practices that let a small team ship fast without chaos.

Underneath that is a habit: research-first, TDD-guided, review-immediately. New code gets a review agent the moment it's written. Long jobs run detached and notify on state change. Then I generalize the whole thing. What starts as personal tooling becomes shared org practice — coding standards, testing, review, security scanning and deployment, all inherited by every engineer's agent.

R&D — off the clock

// personal systems, built & running daily
R&D/01 ● Self-sustaining agent harness

A system that maintains itself

A personal agent-operating layer: purpose-built sub-agents (reviewers, build-error resolvers, architect, planner, security reviewer), custom commands and skills — engineered to be self-healing and self-learning.

  • Dedup-cached security review that escalates to a stronger model only on flagged findings
  • Config-driven release engine — one engine, per-repo spec files
  • Rate-limit auto-resume: queue pending work, drain it headlessly when the window resets
  • A recurring self-audit that re-mines its own history for improvements

R&D/02 ● Privacy-first knowledge loop

A second brain that never phones home

A fully-local pipeline that turns daily work into a curated, self-maintaining knowledge vault — 100% on-device inference, nothing sent to the cloud. The strongest self-directed systems artifact I've built.

  • Scheduled ingest → summarize → promote stages, drafting durable notes automatically
  • Local LLM served on-device behind a thermal / load / RAM guard
  • An LLM-maintained wiki (Karpathy pattern) over a three-layer PARA vault
  • Security-by-design: a secret scrubber redacts keys and tokens before anything touches disk
