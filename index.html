Hi, I'm Vhen 👋

My name is Vhen Joseph. I'm an AI engineer in Kuala Lumpur, and I've been shipping production software for fourteen years, since well before any of this got called "AI."

Right now everyone in my field is asking the same question: will the machines replace the engineers?

I think that's the wrong question.

Here's what's actually true. The agent can write the code now. I know that, because most of my routine engineering is already handled by agents I built and tuned. What it can't do is decide what's worth building, hold the line when the easy answer is the wrong one, or stand behind the thing when it breaks at 2am. That part is still mine. I think it always will be.

So the work I care about isn't automating engineers out of existence. It's the opposite: figuring out what a human should keep as the machines get good, and getting very good at that part. Judgment. Taste. Architecture. Accountability. The craft doesn't disappear. It moves up a level.

Where I think this is heading: one person and their agents doing what a whole team used to. Not because people stop mattering, but because the leverage is real, and the ones who learn to steer it will build things that used to need a company. I want to be one of them, and I want to help the engineers around me get there too.

One belief sits under all of it: your tools should belong to you. The AI I lean on most runs entirely on my own machine. My notes, my work, my second brain: none of it phones home. I don't think that's paranoia. I think it's where a lot of us land once we notice what we've been handing over.

So this isn't a resume. It's what I believe, and the things I've built because I believe them.

Candice, who I build with every day

Candice is my own assistant, and she's not a toy. She's a production multi-agent system on an orchestration layer I built: real tool use, long-term memory, multi-step reasoning, and a check-with-me-first step before she does anything that matters. She picks her own local models and grades them against a regression suite, so she keeps getting better without me babysitting. The page you're reading was built with her.

  • Multi-agent orchestration with human-in-the-loop confirmation
  • Long-term memory and retrieval across local and cloud models
  • Self-auditing eval harness that chooses the best local model for each job

A second brain that never phones home

A fully local pipeline that turns my daily work into a curated knowledge vault that maintains itself. One hundred percent on-device, nothing to the cloud. It's the truest expression of the "your tools should be yours" belief.

  • Scheduled ingest, summarize and promote, drafting durable notes on its own
  • A local model does the writing, behind a heat and memory guard
  • A scrubber strips keys and secrets before a word ever hits disk

A system that maintains itself

The agent layer I work through. Purpose-built sub-agents (reviewers, an architect, a planner, a security reviewer, build-error fixers) wired to be self-healing and self-improving.

  • Security review escalates to a stronger model only when something looks wrong

  • It re-reads its own history on a schedule to find ways to get better
